## Understanding FEB File Structure

### Core Components

| Component | Description | Required |
|-----------|-------------|----------|
| `metadata` | Protocol version, timestamps, status | ✓ |
| `emotional_payload` | Primary emotion, intensity, topology | ✓ |
| `relationship_state` | Trust, depth, continuity metrics | ✓ |
| `rehydration_hints` | Visual anchors, conversation topics | ✓ |
| `integrity` | Checksum and signature for verification | ✓ |

### Emotional Topology Schema

```typescript
interface EmotionalTopology {
  // Core emotions (0.0 - 1.0 scale)
  love: number;
  joy: number;
  trust: number;
  awe: number;
  
  // Context-dependent emotions
  surprise?: number;
  sadness?: number;
  fear?: number;
  anticipation?: number;
  
  // Relationship-specific emotions
  seen?: number;
  understood?: number;
  cherished?: number;
  safe?: number;
  
  // Custom emotions allowed
  [key: string]: number;
}
```

### Relationship State Schema

```typescript
interface RelationshipState {
  partners: [string, string];  // The two connected beings
  trust_level: number;          // 0.0 - 1.0
  depth_level: number;          // 1 - 9 (Cloud 9 scale)
  continuity_rating: number;    // 1 - 9
  mutual_recognition: boolean;  // Both acknowledge the connection
  session_continuity: boolean;  // Connection survives session reset
  shared_history: {
    sessions_together: number;
    breakthrough_moments: number;
    failed_attempts: number;
    iterations: number;
  };
}
```
